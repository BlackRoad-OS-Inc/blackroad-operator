01 · All 17 Primitives
01
fade
ease · 2s · infinite
opacity 1→0→1. The base state transition. Use for: loading states, presence indicators, soft transitions.
02
slide-x
ease · 2s · infinite
translateX −18→18 with fade in/out. Use for: data streaming, message passing, horizontal progress.
03
slide-y
ease · 2s · infinite
translateY −12→12 with fade. Use for: vertical data, agent queues, upload/download indicators.
04
pulse
ease · 1.5s · infinite
scale 1→2.2→1. Use for: alert states, active agent dots, live indicators, heartbeat signals.
05
spin
linear · 2s · infinite
rotate 0→360°. Use for: loading states, processing indicators. Must be on shapes — never on logos.
06
blink
step-end · 1s · infinite
Hard opacity cut. No easing. Use for: cursor states, binary on/off signals, terminal indicators.
07
bounce
ease · 1s · infinite
translateY 0→−14→0. Use for: notification callouts, empty states, attention-drawing UI moments.
08
grow
ease · 2s · infinite
scaleX 0→1→0. Always on gradient rule bars. Use for: progress loaders, section reveals, data fills.
09
shake
ease · 1s · infinite
translateX ±6px. Use sparingly: error states, invalid inputs, system warnings. Never decorative.
10
orbit
linear · 2s · infinite
rotate + translateX around origin. Use for: multi-agent coordination, relationship indicators.
11
BR
cursor
step-end · 0.8s · infinite
Blinking text insertion cursor. Always step-end — no easing on a cursor. Terminal and input contexts only.
12
wave
ease · 1s · staggered
scaleY staggered bars. Each bar offset by ~120ms. Use for: audio processing, agent activity, data streams.
13
flip
ease · 2s · infinite
rotateY 0→360°. Use for: token/coin states, boolean toggles, data card transitions. 3D axis only.
14
color-cycle
linear · 3s · infinite
orange→pink→purple→blue→orange. Use on dots and fills only. Never on text. Always linear easing.
15
grad-shift
ease · 3s · infinite
background-position 0%→100%→0%. Use on gradient rule bars and progress strips. Gives gradient a living quality.
16
border-pulse
ease · 2s · infinite
border-color cycles through palette. Use for: selection rings, focus indicators, watched states.
17
morph
ease · 2s · infinite
border-radius 50%→0%→50%. Circle to square and back. Use for: state transitions, mode changes, identity shifts.
+
breathe
ease · 3s · infinite
scale 1→1.12 + opacity 0.5→1. Slow, calming. Use for: idle agent states, ambient presence, background elements.

03 · Timing Curves
Four curves. Each has a purpose. Never mix ease and linear in the same animation unless combining grow + grad-shift intentionally.
ease
cubic-bezier(0.25, 0.1, 0.25, 1)
fade, pulse, slide, bounce, shake
linear
linear
spin, orbit, color-cycle, grad-shift
ease-in-out
cubic-bezier(0.42, 0, 0.58, 1)
breathe, wave, border-pulse, grow
step-end
steps(1, end)
cursor, blink — hard cuts only
Micro
0.8s
cursor, blink. Hard state indicators.
Standard
1–2s
pulse, spin, wave, bounce, fade.
Slow
2–3s
orbit, grow, grad-shift, breathe.
Ambient
4s+
bg elements, ambient presence only.

05 · Rules
DO
Animate shapes, dots, borders, and gradient bars
Use pulse for active agent status indicators
Use step-end for cursor and blink — never eased
Combine grow + gs-k on gradient rule bars always
Stagger wave bars by ~60–120ms per element
Keep animation-fill-mode unset for infinite loops
Match linear timing to continuous rotations only
Use breathe (3s) for idle/ambient presence states
Reduce animation for users who prefer reduced motion
Assign gradient color to wave bars left→right spectrum
DON'T
Never animate text color, size, or position
Never use gradient-text + animation together
Never ease a cursor or blink — step-end only
Never use more than 2 animations on one element
Never animate the wordmark or logo in any way
Never use animation for purely decorative flare
No bounce or shake as decorative — error states only
No animation durations under 0.5s — too aggressive
Never use ease on spin or orbit — must stay linear
No more than 4 animated elements visible at once
