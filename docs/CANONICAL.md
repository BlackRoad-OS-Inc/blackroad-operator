# Canonical Truth

> This repository is the **source of truth** for BlackRoad OS operations, governance wiring, and ecosystem coordination.

## Doctrine

Canonical decisions and system-wide architecture live here. This is where the "why" and "how" of BlackRoad OS are recorded and maintained.

### What Belongs Here

| Type | Location | Purpose |
|------|----------|---------|
| Architecture decisions | `docs/` | Finalized system design |
| Org topology | `docs/org-map.md` | Organization structure |
| Domain inventory | `docs/domains.md` | Cloudflare domain mapping |
| Marketing language | `docs/marketing/copy.md` | Approved messaging |
| Operator contract | `docs/operator-contract.md` | Operational guarantees |

### Rules

1. **Discussions** are for doctrine, architecture, and governance proposals
2. **Issues** are for actionable work items with clear acceptance criteria
3. **Committed docs** are for finalized, approved content
4. **PRs** must reference the issue or discussion they resolve

### Labels

| Label | Purpose |
|-------|---------|
| `dns` | Domain and DNS configuration |
| `operator` | Operator tooling and CLI |
| `org-sync` | Cross-organization synchronization |
| `marketing` | Brand, copy, and messaging |
| `infra` | Infrastructure and deployment |
| `agents` | AI agent system |
| `governance` | Governance and compliance |

## Canonical Threads

Use GitHub Discussions for long-lived architectural topics:

- **Canonical** -- System architecture, design decisions
- **Governance** -- Policies, access control, compliance
- **Infra & Mesh** -- Infrastructure topology, networking
- **Operator** -- CLI tooling, automation
- **Marketing & Copy** -- Brand voice, messaging
- **Q&A** -- General questions
